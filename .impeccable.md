# FairLens Design Context

## Users & Purpose

**Target Audience:** ML Engineers and Compliance Officers who need to:
- Upload model prediction data (CSV)
- Run automated statistical fairness tests
- Receive audit-ready verdicts

**Context:** Data-dense professional tool requiring precision and trustworthiness. Users are technical and need to quickly interpret results to take action.

**Job to be Done:** Fairness auditing of ML models - upload predictions, define protected attributes, run metrics, get pass/fail verdicts for compliance reporting.

---

**Voice:** Professional, authoritative, precision instrument
**Tone:** Trustworthy, efficient, no-nonsense

## Design Principles

1. **Data-first:** Interface recedes so data stands out
2. **Professional precision:** Clean, purposeful, authoritative - no decoration for decoration's sake
3. **Trust through clarity:** Verdicts must be instantly interpretable with clear pass/fail/warning states
4. **Efficient workflow:** Minimize clicks, clear progress indicators, optimistic UI
5. **Accessibility by default:** All semantic colors pass WCAG AA (4.5:1 contrast)
